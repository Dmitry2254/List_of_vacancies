## 2. Описание предметной области

Предметная область "Список вакансий" предназначена для процесса найма и централизованного хранения данных о доступных позициях в организации. 
Система позволит HR-менеджеру создавать вакансии, указывать требования к кандидатам, должностные обязанности и условия труда.
Пользователь может просматривать список доступных вакансий, фильтровать их по параметрам и просматривать описание каждой из них.
Администратор обладает полным доступом к системе. 
Решение ориентировано на учебное моделирование работы простой ИС без серверной части.
---

## 3. Пользователи системы

| Роль пользователя | Описание функций |
|------------------|------------------|
| Администратор |Управление учетными записями (создание, удаление), контроль доступа к системе |
| HR-менеджер |Создание и редактирвоание вакансий, изменение статусов (открыта/закрыта), работа с архивом объявлений |
| Пользователь |Просмотр списка всех актуальных вакансий, поиск и фильтрация по параметрам, просмотр детальной информации |

## 4. Основные сущности предметной области (данные)

| Сущность | Описание | Основные атрибуты |
|--------|----------|-------------------|
| Вакансия | Объявление о поиске сотрудника, созданное HR-менеджером|Id, заголовок, описание, требования, запрлата, локация, опыт, тип занятости |
| Пользователь | Участники системы (Администратор, R-менеджер и пользователь) | Id, ФИО, email, пароль, роль|
| Отклик |Факт выражения интереса пользователя к конкретной вакансии |id, вакансия_id, пользователь_id, дата отклика, статус |

> Допускается начинать с 1–2 сущностей и расширять модель в ходе выполнения лабораторных работ.
